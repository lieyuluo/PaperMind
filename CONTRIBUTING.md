# Contributing

## Workflow

1. Create an issue or task before starting significant work.
2. Create a branch from `main`.
3. Keep changes focused and reviewable.
4. Open a pull request.
5. Wait for review and CI before merging.

## Commit Messages

Use Conventional Commits:

```text
feat: add user invitation flow
fix: handle expired session token
docs: update deployment guide
refactor: simplify permission check
test: add billing service tests
```

## Pull Request Checklist

- The change has a clear purpose.
- Tests were added or updated where useful.
- Documentation was updated when behavior changed.
- No secrets, credentials, or local-only files are committed.
- The change has been tested locally.
